# 婚禮賓客管理系統

婚宴當天使用的賓客管理工具，支援男女方同時在不同裝置上操作。

## 功能

- 男方／女方分流，兩人可同時用不同手機操作
- 記錄禮金金額（點擊直接編輯）
- 追蹤禮餅領取狀態
- 標記未出席賓客
- 依姓名、桌次、禮餅狀態篩選
- CSV 批次匯入名單

## 技術

- **前端**：Vue 3 + Vite + Pinia
- **資料庫**：Supabase（PostgreSQL）
- **部署**：Vercel

## 本機開發

1. 安裝依賴

```bash
npm install
```

2. 建立 `.env.local`，填入 Supabase 設定

```
VITE_SUPABASE_URL=你的_Project_URL
VITE_SUPABASE_ANON_KEY=你的_anon_key
```

3. 啟動開發伺服器

```bash
npm run dev
```

## Supabase 資料表設定

在 Supabase SQL Editor 執行：

```sql
create table guests (
  id bigint generated by default as identity primary key,
  name text not null,
  side text not null check (side in ('groom', 'bride')),
  table_number text,
  needs_cake boolean default false,
  cake_received boolean default false,
  gift_money integer default 0,
  absent boolean default false
);

alter table guests enable row level security;
create policy "allow all" on guests for all using (true) with check (true);
```

## 部署到 Vercel

1. 將專案推到 GitHub
2. 在 [Vercel](https://vercel.com) import repo
3. 設定環境變數 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`
4. Deploy
